Finding optimal connectivity paths and minimum spanning weights are fundamental optimization tasks. This module covers shortest paths over positive, negative, and bounded integer weights, and investigates undirected spanning forests alongside Directed MSTs.

---

## 1. Shortest Path Algorithms

### A. Dijkstra's Algorithm
Finds the single-source shortest path in graphs with non-negative edge weights.
* **Sparse Heap Variant**: $O(E \log V)$ using `std::priority_queue`.
* **Dense Array Variant**: $O(V^2)$ by scanning all nodes to find the minimum distance vertex. Highly optimized in dense graphs where $E \approx V^2$.

### B. Bellman-Ford & SPFA (Shortest Path Faster Algorithm)
* **Bellman-Ford**: Relaxes all edges $V-1$ times in $O(VE)$ to handle negative edge weights and detect negative cycles.
* **SPFA**: A queue-based optimization of Bellman-Ford that only relaxes edges from updated nodes.
  * **Average Complexity**: $O(E)$
  * **SLF/LLL Optimizations**: Small Label First (insert at front of deque if $d(v) < d(\text{front})$) speeds up convergence.

<Callout type="warning" title="SPFA Worst Case is Hackable!">
The worst-case complexity of SPFA remains $O(V E)$. On grid graphs or graphs containing nested chains/cycles, a clever testmaker can easily force your queue to overflow and time out. Never use SPFA unless negative weights prevent you from using Dijkstra!
</Callout>

### C. Dial's Algorithm ($O(V + W)$)
When edge weights are small integers bounded by $W$:
* Instead of a log-heap, we maintain an array of $W + 1$ buckets (represented as lists of vertices).
* We scan the buckets sequentially to extract the minimum distance node. This achieves a linear runtime!

### D. Floyd-Warshall ($O(V^3)$)
Computes all-pairs shortest paths using dynamic programming:
$$dp[i][j][k] = \min(dp[i][j][k-1], dp[i][k][k-1] + dp[k][j][k-1])$$

## 2. Minimum Spanning Tree (MST)

### A. Kruskal's Algorithm ($O(E \log E)$)
Sorts all edges by weight and greedily inserts them using Disjoint Set Union (DSU) to avoid forming cycles.

### B. Prim's Algorithm ($O(E \log V)$)
Greedily expands a single tree node-by-node using a priority queue of cut-edges.

### C. Boruvka's Algorithm ($O(E \log V)$)
Merges components in parallel. In each round, every connected component identifies its cheapest outgoing edge. These edges are added to the MST simultaneously.

<Callout type="tip" title="Why Boruvka is Crucial in CP">
Since Boruvka's algorithm halves the number of components in each round, it terminates in at most $\log V$ rounds. This is extremely powerful in problems where edges are not explicit (e.g. finding the XOR MST of a set of values, where we can find components' nearest neighbors using a **Trie** in $O(N \log (\max A))$ total time).
</Callout>

## 3. Directed MST (Chu-Liu-Edmonds Algorithm)
Given a directed graph, find a spanning tree directed away from root $r$ with minimum total weight.
* Kruskal/Prim does not work because directed cycles cannot be easily resolved greedily.
* **Chu-Liu-Edmonds Algorithm**:
  1. For each node $u \neq r$, pick its cheapest incoming edge $(v \to u)$ with weight $W$.
  2. If the chosen edges form no cycles, we are done! They constitute the Directed MST.
  3. If there are cycles, contract each cycle into a virtual node.
  4. For all outgoing/incoming edges of the cycle, adjust their weights: an incoming edge $(x \to u)$ in the cycle has its weight modified to $W_{new} = W_{old} - W_{\text{cycle-in}}$, compensating for breaking the cycle edge.
  5. Recursively solve for the contracted graph, and unroll the cycles.

### C++ Chu-Liu-Edmonds directed MST Implementation:
```cpp
struct Edge {
    int u, v; ll w;
};

ll directed_mst(int N, int M, int root, vector<Edge>& edges) {
    ll ans = 0;
    vector<int> parent(N), comp(N), bio(N);
    vector<ll> min_in(N);

    while (true) {
        fill(min_in.begin(), min_in.end(), INF);
        fill(parent.begin(), parent.end(), -1);
        
        for (auto& e : edges) {
            if (e.u != e.v && e.w < min_in[e.v]) {
                min_in[e.v] = e.w;
                parent[e.v] = e.u;
            }
        }
        
        // Check if all nodes (except root) have an incoming edge
        for (int i = 0; i < N; i++) {
            if (i == root) continue;
            if (min_in[i] == INF) return -1; // No MST possible
        }

        int scc_cnt = 0;
        fill(comp.begin(), comp.end(), -1);
        fill(bio.begin(), bio.end(), -1);
        
        // Cycle detection
        for (int i = 0; i < N; i++) {
            if (i == root) continue;
            ans += min_in[i];
            
            int curr = i;
            while (curr != root && bio[curr] != i && comp[curr] == -1) {
                bio[curr] = i;
                curr = parent[curr];
            }
            // Cycle found!
            if (curr != root && comp[curr] == -1) {
                int u = curr;
                do {
                    comp[u] = scc_cnt;
                    u = parent[u];
                } while (u != curr);
                scc_cnt++;
            }
        }

        if (scc_cnt == 0) break; // No cycles left!

        // Compress other vertices
        for (int i = 0; i < N; i++) {
            if (comp[i] == -1) comp[i] = scc_cnt++;
        }

        // Adjust edges for contracted vertices
        for (auto& e : edges) {
            ll prev_in = min_in[e.v];
            e.u = comp[e.u];
            e.v = comp[e.v];
            if (e.u != e.v) {
                e.w -= prev_in; // Adjust weight
            }
        }
        N = scc_cnt;
        root = comp[root];
    }
    return ans;
}
```
