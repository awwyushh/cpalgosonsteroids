## Why This Topic Matters

CRT reconstructs a unique integer from its residues modulo pairwise coprime moduli. Essential for:
- Combining answers computed mod different primes (avoid 64-bit overflow)
- Solving systems of congruences
- NTT with multiple primes for large convolution
- Competitive math problems with "find x ≡ a₁ (mod m₁), x ≡ a₂ (mod m₂)"

**Contest relevance**: CF Div.1 C–D, NTT-heavy problems, any system of congruences.

## Learning Objectives

- State and apply CRT for coprime moduli

## CRT Statement

**Theorem**: Given pairwise coprime $m_1, m_2, \ldots, m_k$ and any $a_1, \ldots, a_k$, there exists a unique $x \in [0, M)$ where $M = \prod m_i$ such that:

$$x \equiv a_i \pmod{m_i} \quad \forall i$$

---

## Two-Moduli Case

Combine $x \equiv a \pmod{m_1}$ and $x \equiv b \pmod{m_2}$ where $\gcd(m_1, m_2) = 1$:

```cpp
// Returns (value, modulus) of combined congruence
// x ≡ a (mod m1), x ≡ b (mod m2)
pair<long long, long long> crt2(long long a, long long m1, long long b, long long m2) {
    long long x, y;
    long long g = ext_gcd(m1, m2, x, y);
    // g must be 1 (coprime)
    long long lcm = m1 * m2;  // use __int128 if large
    long long result = ((__int128)a * m2 % lcm * y % lcm + (__int128)b * m1 % lcm * x % lcm) % lcm;
    result = (result % lcm + lcm) % lcm;
    return {result, lcm};
}
```

## General CRT (Iterative Combination)

Works even for non-coprime moduli if a solution exists:

```cpp
// Solve system x ≡ r[i] (mod m[i])
// Returns {-1, -1} if no solution
pair<long long, long long> crt(vector<long long> r, vector<long long> m) {
    long long cur_r = 0, cur_m = 1;
    for (int i = 0; i < (int)r.size(); i++) {
        long long g = __gcd(cur_m, m[i]);
        if ((r[i] - cur_r) % g != 0) return {-1, -1};
        long long lcm = cur_m / g * m[i];
        long long diff = (r[i] - cur_r) / g;
        long long inv = modinv(cur_m / g, m[i] / g);
        cur_r = (cur_r + cur_m * (diff % (m[i]/g) * inv % (m[i]/g))) % lcm;
        cur_m = lcm;
    }
    return {(cur_r % cur_m + cur_m) % cur_m, cur_m};
}
```

## Garner's Algorithm

Represent a large number $x$ as mixed-radix with moduli $m_1, \ldots, m_k$:

$$x = a_1 + a_2 m_1 + a_3 m_1 m_2 + \cdots$$

Compute the mixed-radix digits $a_i$ without computing $x$ directly (which may be huge):

```cpp
// Garner's algorithm: recover x mod final_mod
// x ≡ r[i] (mod p[i]) where p[i] are distinct primes
long long garner(vector<long long> r, vector<long long> p, long long mod) {
    int n = r.size();
    // coefficients a[i] in mixed-radix representation
    vector<long long> a(r);
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < i; j++) {
            // a[i] = (a[i] - a[j]) * modinv(p[j], p[i]) mod p[i]
            a[i] = (a[i] - a[j]) % p[i];
            a[i] = a[i] * modinv(p[j] % p[i], p[i]) % p[i];
            a[i] = (a[i] + p[i]) % p[i];
        }
    }
    // Reconstruct x mod final_mod
    long long result = 0, prod = 1;
    for (int i = 0; i < n; i++) {
        result = (result + a[i] % mod * prod) % mod;
        prod = prod * (p[i] % mod) % mod;
    }
    return result;
}
```

**Use case**: NTT with 3 primes ($p_1 \approx p_2 \approx p_3 \approx 10^9$), then Garner to recover result mod $10^9+7$.

## Common Mistakes

1. **Overflow in $m_1 \cdot m_2$**: use `__int128` or check before multiplying.
2. **Non-coprime moduli**: standard CRT fails; use general iterative version.
3. **Negative residues**: normalize with `(r % m + m) % m`.
