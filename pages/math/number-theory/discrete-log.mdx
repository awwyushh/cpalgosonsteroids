## Why This Topic Matters

Discrete logarithm (DLOG): find $x$ such that $a^x \equiv b \pmod{p}$. Baby-step Giant-step solves it in $O(\sqrt{p})$ time and space. Pohlig-Hellman reduces to smaller subgroups when $p-1$ is smooth.

**Contest relevance**: CF Div.1 D/E, cryptography-flavored CP problems.

---

## Learning Objectives

- Understand the DLOG problem and why it's hard in general
- Implement Baby-step Giant-step (BSGS) in $O(\sqrt{p})$
- Apply Pohlig-Hellman for smooth order groups
- Handle the case where $a$ is not a generator

## Baby-Step Giant-Step (BSGS)

Solve $a^x \equiv b \pmod{p}$ for $0 \leq x < p-1$.

Let $m = \lceil\sqrt{p}\rceil$. Write $x = im - j$ for $0 \leq i, j < m$.

Then $a^{im} \equiv b \cdot a^j \pmod{p}$.

1. **Baby steps**: precompute $b \cdot a^j \bmod p$ for $j = 0, 1, \ldots, m-1$. Store in hash map.
2. **Giant steps**: for each $i = 1, 2, \ldots, m$, compute $a^{im} \bmod p$ and look up in table.

```cpp
long long baby_giant(long long a, long long b, long long p) {
    long long m = (long long)ceil(sqrt(p)) + 1;
    unordered_map<long long, long long> table;
    
    // Baby steps: b * a^j mod p
    long long val = b % p;
    for (long long j = 0; j < m; j++) {
        table[val] = j;
        val = val * a % p;
    }
    
    // Giant steps: a^(im) mod p
    long long am = power(a, m, p);
    val = 1;
    for (long long i = 1; i <= m; i++) {
        val = val * am % p;
        if (table.count(val)) {
            long long x = i * m - table[val];
            return (x % (p - 1) + (p - 1)) % (p - 1);
        }
    }
    return -1;  // no solution
}
```

<Callout type="info">
BSGS requires $p$ prime and $\gcd(a, p) = 1$. For general moduli, use the extended BSGS variant.
</Callout>

## Generalized BSGS (Non-Prime Modulus)

For $a^x \equiv b \pmod{n}$ with general $n$ (not prime):

Reduce to: let $g = \gcd(a, n)$. If $g \nmid b$, no solution.

Divide: $(a/g)^x \equiv (b/g) \pmod{n/g}$ — repeat until $\gcd(a, n/g^k) = 1$, then apply BSGS.

```cpp
// Generalized BSGS: solve a^x = b mod n
long long gen_bsgs(long long a, long long b, long long n) {
    if (b == 1 || n == 1) return 0;
    long long cnt = 0, t = 1;
    for (long long g = __gcd(a, n); g != 1; g = __gcd(a, n)) {
        if (b % g != 0) return -1;
        n /= g; b /= g;
        t = t * (a / g) % n;
        cnt++;
        if (t == b) return cnt;
    }
    // Now gcd(a, n) = 1, apply BSGS on a^x * t ≡ b (mod n)
    long long m = (long long)ceil(sqrt(n)) + 1;
    unordered_map<long long, long long> table;
    long long val = b;
    for (long long j = 0; j < m; j++) {
        table[val] = j;
        val = val * a % n;
    }
    long long am = power(a, m, n);
    val = t;
    for (long long i = 1; i <= m + 1; i++) {
        val = val * am % n;
        if (table.count(val))
            return i * m - table[val] + cnt;
    }
    return -1;
}
```

## Pohlig-Hellman

When $p - 1$ factors as $p_1^{e_1} \cdots p_k^{e_k}$ (smooth $p-1$), use CRT to combine subgroup solutions.

**Key idea**: solve $a^x \equiv b \pmod{p}$ in each subgroup of order $p_i^{e_i}$, then CRT.

**Complexity**: $O(\sum e_i (\log p_i + \sqrt{p_i}))$ — much faster when all prime factors of $p-1$ are small.

## Common Mistakes

1. **Off-by-one**: $x = im - j$ range; ensure $i$ goes up to $m$, not $m-1$.
2. **No solution case**: BSGS returns $-1$; handle when $a$ is not a primitive root.
3. **Modular issues**: use `(x % (p-1) + (p-1)) % (p-1)` to normalize.
