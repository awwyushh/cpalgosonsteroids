## Why This Topic Matters

FFT enables polynomial multiplication in $O(n \log n)$ instead of $O(n^2)$. This unlocks:
- Convolutions (string matching, counting problems)
- Long multiplication of big integers
- Many counting DP optimizations
- NTT for exact integer arithmetic

**Contest relevance**: Codeforces Div.1 E/F, AtCoder ARC/AGC D/E. Any "count pairs with sum $k$" or convolution-style problem.

## Representation of Polynomials

A degree-$n$ polynomial $A(x) = a_0 + a_1 x + \cdots + a_n x^n$ has two representations:

1. **Coefficient form**: $(a_0, a_1, \ldots, a_n)$ — what we usually work with
2. **Value form**: $(A(x_0), A(x_1), \ldots, A(x_n))$ for $n+1$ distinct $x_i$ — uniquely determines $A$

| Operation | Coefficient form | Value form |
|-----------|-----------------|------------|
| Evaluate at $x$ | $O(n)$ | $O(1)$ if $x \in \{x_i\}$ |
| Multiply $A \cdot B$ | $O(n^2)$ | $O(n)$ pointwise |
| Add $A + B$ | $O(n)$ | $O(n)$ |

**Key insight**: Multiplication is $O(n)$ in value form. If we can convert back and forth in $O(n \log n)$, polynomial multiplication costs $O(n \log n)$.

**DFT** = convert coefficient → value at special points (roots of unity).

**IDFT** = convert value → coefficient.

---

## Discrete Fourier Transform (DFT)

Evaluate $A(x)$ at the $n$-th roots of unity: $\omega_n^0, \omega_n^1, \ldots, \omega_n^{n-1}$ where:

$$\omega_n = e^{2\pi i/n} = \cos(2\pi/n) + i\sin(2\pi/n)$$

The DFT of $(a_0, \ldots, a_{n-1})$ is $(A(\omega_n^0), A(\omega_n^1), \ldots, A(\omega_n^{n-1}))$.

$$\hat{a}_k = \sum_{j=0}^{n-1} a_j \omega_n^{jk}$$

The inverse DFT:

$$a_j = \frac{1}{n} \sum_{k=0}^{n-1} \hat{a}_k \omega_n^{-jk}$$

---

## The Cooley-Tukey FFT Algorithm

**Idea**: Divide the polynomial into even and odd indexed coefficients.

$$A(x) = A_{\text{even}}(x^2) + x \cdot A_{\text{odd}}(x^2)$$

where $A_{\text{even}}(x) = a_0 + a_2 x + a_4 x^2 + \cdots$ and $A_{\text{odd}}(x) = a_1 + a_3 x + a_5 x^2 + \cdots$.

To evaluate at $\omega_n^k$ and $\omega_n^{k+n/2}$:

$$A(\omega_n^k) = A_{\text{even}}(\omega_{n/2}^k) + \omega_n^k \cdot A_{\text{odd}}(\omega_{n/2}^k)$$
$$A(\omega_n^{k+n/2}) = A_{\text{even}}(\omega_{n/2}^k) - \omega_n^k \cdot A_{\text{odd}}(\omega_{n/2}^k)$$

This is the **butterfly** operation. Recurse: $T(n) = 2T(n/2) + O(n) \Rightarrow T(n) = O(n \log n)$.

## Implementation

```cpp
#include <complex>
#include <vector>
using namespace std;
typedef complex<double> cd;
const double PI = acos(-1.0);

void fft(vector<cd>& a, bool invert) {
    int n = a.size();
    if (n == 1) return;

    // Bit-reversal permutation
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }

    // Butterfly operations
    for (int len = 2; len <= n; len <<= 1) {
        double ang = 2 * PI / len * (invert ? -1 : 1);
        cd wlen(cos(ang), sin(ang));
        for (int i = 0; i < n; i += len) {
            cd w(1);
            for (int j = 0; j < len / 2; j++) {
                cd u = a[i+j], v = a[i+j+len/2] * w;
                a[i+j] = u + v;
                a[i+j+len/2] = u - v;
                w *= wlen;
            }
        }
    }

    if (invert) {
        for (cd& x : a) x /= n;
    }
}

// Multiply polynomials A and B
vector<long long> multiply(vector<int>& a, vector<int>& b) {
    vector<cd> fa(a.begin(), a.end()), fb(b.begin(), b.end());
    int n = 1;
    while (n < (int)(a.size() + b.size())) n <<= 1;
    fa.resize(n); fb.resize(n);

    fft(fa, false);
    fft(fb, false);
    for (int i = 0; i < n; i++) fa[i] *= fb[i];
    fft(fa, true);

    vector<long long> result(n);
    for (int i = 0; i < n; i++)
        result[i] = llround(fa[i].real());
    return result;
}
```

## Precision Issues

FFT with `double` has limited precision (~15 decimal digits). For large coefficients:

- Coefficients up to $10^4$, result up to $10^{12}$: double is fine.
- Larger: use `long double` or NTT (exact).

**Split technique** for large coefficients: split each coefficient $a_i = A_i + B_i \cdot 2^{15}$ and compute three FFTs:

```cpp
// Avoids precision loss for large integer coefficients
vector<long long> multiply_large(vector<long long>& a, vector<long long>& b) {
    int n = 1;
    while (n < (int)(a.size() + b.size())) n <<= 1;
    vector<cd> fa(n), fb(n);
    for (int i = 0; i < (int)a.size(); i++) {
        fa[i] = {(double)(a[i] & ((1 << 15) - 1)), (double)(a[i] >> 15)};
    }
    for (int i = 0; i < (int)b.size(); i++) {
        fb[i] = {(double)(b[i] & ((1 << 15) - 1)), (double)(b[i] >> 15)};
    }
    fft(fa, false); fft(fb, false);
    // ... combine appropriately
    // Full implementation: 3 FFTs trick
}
```

## Applications

### String Matching with Wildcards

Pattern $P$ with wildcards matches position $i$ of text $T$ iff:

$$\sum_j P[j] \cdot (P[j] - T[i+j])^2 = 0$$

This is a convolution!

### Counting Pairs with Sum $k$

Given arrays $A$ and $B$, count pairs $(a, b)$ with $a + b = k$:
- Build frequency arrays $f_A[v]$ = # of elements equal to $v$ in $A$.
- Similarly $f_B$.
- Answer = $(f_A \ast f_B)[k]$ (convolution).

### Subset Sum (counting)

Given a set, count how many subsets have sum exactly $k$:
- This is $[x^k]$ in the product $\prod_i (1 + x^{a_i})$.
- FFT the polynomial (or use NTT).

## Common Mistakes

1. **Size must be power of 2**: pad to next power of 2.
2. **Size must be $\geq \deg(A) + \deg(B) + 1$**: otherwise wrap-around.
3. **Rounding**: use `llround` not `(long long)` for rounding.
4. **Precision**: for large values, use `long double` or NTT.
5. **Invert flag wrong**: DFT uses $\omega_n$, IDFT uses $\omega_n^{-1}$ and divides by $n$.
