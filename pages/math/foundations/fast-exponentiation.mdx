## Why This Topic Matters

Computing $a^n$ naively takes $O(n)$ multiplications — useless for $n \leq 10^{18}$. Binary exponentiation does it in $O(\log n)$ using repeated squaring. It also generalizes to matrix exponentiation, polynomial exponentiation, and any associative operation.

**Contest relevance**: Required for modular inverse (Fermat), matrix recurrences, counting paths, almost every number theory problem.

---

## Learning Objectives

- Compute $a^n \bmod m$ in $O(\log n)$
- Understand the repeated squaring idea
- Implement iterative and recursive versions
- Apply to matrix exponentiation
- Handle edge cases: $n=0$, $m=1$

## Repeated Squaring

Key idea: $a^n = \begin{cases} (a^{n/2})^2 & \text{if } n \text{ even} \\ a \cdot (a^{n/2})^2 & \text{if } n \text{ odd} \end{cases}$

Or equivalently, decompose $n$ in binary: $a^{13} = a^{1101_2} = a^8 \cdot a^4 \cdot a^1$.

**Recursive**:
```cpp
long long power(long long a, long long n, long long mod) {
    if (n == 0) return 1;
    long long half = power(a, n / 2, mod);
    long long result = half * half % mod;
    if (n % 2) result = result * a % mod;
    return result;
}
```

**Iterative** (preferred in CP — no stack overhead):
```cpp
long long power(long long a, long long n, long long mod) {
    long long result = 1;
    a %= mod;
    while (n > 0) {
        if (n & 1) result = result * a % mod;
        a = a * a % mod;
        n >>= 1;
    }
    return result;
}
```

<Callout type="info">
Iterative version processes bits of `n` from LSB to MSB. At each step: if current bit is 1, multiply result by current power of a.
</Callout>

## Safe Multiplication for Large Moduli

When $\text{mod} > 2^{31}$, `a * a` can overflow `long long` before `% mod`. Two solutions:

```cpp
// Option 1: __int128 (preferred, available on GCC)
long long power(long long a, long long n, long long mod) {
    long long result = 1;
    a %= mod;
    while (n > 0) {
        if (n & 1) result = (__int128)result * a % mod;
        a = (__int128)a * a % mod;
        n >>= 1;
    }
    return result;
}

// Option 2: mulmod via binary method (portable but slower)
long long mulmod(long long a, long long b, long long mod) {
    long long result = 0;
    a %= mod;
    while (b > 0) {
        if (b & 1) result = (result + a) % mod;
        a = (a * 2) % mod;
        b >>= 1;
    }
    return result;
}
```

<Callout type="info">
For Miller-Rabin and Pollard's rho, mod can be up to $10^{18}$ — always use `__int128` or `mulmod`.
</Callout>

## Without Modulo

For large $n$ without modular reduction (e.g., counting purposes using `__int128` or big integers):

```cpp
long long power(long long a, long long n) {
    long long result = 1;
    while (n > 0) {
        if (n & 1) result *= a;
        a *= a;
        n >>= 1;
    }
    return result;
}
```

## Generalization: Any Associative Operation

Binary exponentiation works for any monoid $(S, \cdot, e)$ where $\cdot$ is associative and $e$ is the identity:

```cpp
template<typename T>
T power(T a, long long n, T identity, function<T(T,T)> op) {
    T result = identity;
    while (n > 0) {
        if (n & 1) result = op(result, a);
        a = op(a, a);
        n >>= 1;
    }
    return result;
}
// Works for: integers (mod m), matrices, polynomials, strings (hashing), etc.
```

## Matrix Exponentiation

Any associative operation generalizes. Matrix power for linear recurrences:

```cpp
using Matrix = vector<vector<long long>>;
const long long MOD = 1e9 + 7;

Matrix multiply(const Matrix& A, const Matrix& B) {
    int n = A.size();
    Matrix C(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++)
        for (int k = 0; k < n; k++) if (A[i][k])
            for (int j = 0; j < n; j++)
                C[i][j] = (C[i][j] + A[i][k] % MOD * B[k][j]) % MOD;
    return C;
}

Matrix mat_power(Matrix A, long long n) {
    int sz = A.size();
    Matrix result(sz, vector<long long>(sz, 0));
    for (int i = 0; i < sz; i++) result[i][i] = 1;  // identity
    while (n > 0) {
        if (n & 1) result = multiply(result, A);
        A = multiply(A, A);
        n >>= 1;
    }
    return result;
}
```

**Use case**: $n$-th Fibonacci in $O(\log n)$ via $\begin{pmatrix}F_{n+1}\\F_n\end{pmatrix} = \begin{pmatrix}1&1\\1&0\end{pmatrix}^n \begin{pmatrix}1\\0\end{pmatrix}$.

## Edge Cases

```cpp
// n = 0: a^0 = 1 (even a^0 = 1 by convention)
// mod = 1: result is always 0
// a = 0, n = 0: conventionally 1, but check problem statement
power(0, 0, mod);  // returns 1 in the implementation above
```

<Callout type="warning">
If `mod = 1`, every operation yields 0. The implementation handles this correctly since `1 % 1 = 0`.
</Callout>

## Common Patterns

1. **Modular inverse via Fermat**: `power(a, mod-2, mod)` when mod is prime
2. **Fibonacci in O(log n)**: matrix exponentiation
3. **Count paths of length k**: matrix power of adjacency matrix
4. **Linear recurrence n-th term**: companion matrix approach
5. **Large exponent in exponent**: `power(a, power(b, c, phi), mod)` — use Euler's theorem

---

## Common Mistakes

1. **Intermediate overflow**: `a * a` can overflow `long long` before `% mod`. Cast to `__int128` or ensure `a < mod` before squaring.
2. **Wrong base case**: `n=0` should return 1, not 0.
3. **Forgetting `a %= mod`** at the start — if `a >= mod`, `a*a` overflows.
