# Persistence

Persistent data structures maintain all historical versions. Query any past state in $O(\log n)$.

**Core technique**: **path copying** — when updating, copy only the $O(\log n)$ nodes on the update path.

## Path Copying Principle

When updating node $v$ in version $k$:

1. Create a **new node** $v'$ with the same data as $v$
2. Modify $v'$ with the update
3. Create new nodes for all ancestors of $v$ on the path to root, pointing to updated children
4. Store the new root as version $k+1$

**Cost**: $O(\log n)$ new nodes per update. Total space: $O(n + q \log n)$.

```
Version 1 root → ... → node v (shared)
                          ↑
Version 2 root → ... → node v' (new)
```
